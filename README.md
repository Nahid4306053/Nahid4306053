## 👨‍💻 About Me

Hi, I'm MD Nahid Hasan — a Full-Stack Software Engineer specializing in Banking & Trade Finance Systems.

I currently work at Spectrum Software & Consulting Ltd., contributing to a production trade-finance banking platform deployed for Prime Bank. The platform supports Letter of Credit (LC), Shipping Guarantee (SG), Telegraphic Transfer (TT), Cash Against Documents (CAD), and Acceptance workflows while processing thousands of business-critical transactions each month.

My responsibilities span backend engineering, frontend architecture, infrastructure, DevOps, security, database design, production support, and operational troubleshooting.

Unlike a traditional frontend-focused full-stack role, my work extends deep into authentication systems, infrastructure management, security implementation, observability, and banking workflow automation.

## 🏦 Banking Platform Ownership

### Letter of Credit (LC) Module

Primary contributor responsible for ongoing feature development and architecture improvements.

Key contributions include:

* Complex Redux + Thunk workflow management
* Reusable component architecture
* Performance optimization using useMemo, useCallback, custom hooks, and render optimization techniques
* In-app collaboration/chat functionality
* Automated clause generation
* Template-based LC creation
* Charge and margin calculation workflows

### User Management Module (Built End-to-End)

Designed and implemented the entire user management ecosystem including:

* 10+ configurable banking roles
* RBAC permission management
* Maker-checker approval workflows
* Audit logging
* Login/logout tracking
* Account lockout mechanisms
* Password policy enforcement
* Session lifecycle management
* JWT + OTP hybrid authentication
* Rate limiting and abuse prevention

Also led a significant live-system migration that replaced customer_id with group_id across the platform while maintaining customer-history traceability through versioned snapshots.

### Authentication & Security

Responsible for a substantial portion of authentication and security implementation, including:

* JWT authentication
* OTP verification
* Session-based authentication
* Replay attack prevention
* Cryptographic nonce validation
* URL and payload signature verification
* Content Security Policy implementation
* Security header hardening
* Payload validation
* Vulnerability remediation
* CVE mitigation
* WAF integration support

Collaborated closely with security teams during F5 WAF policy implementation and OpenAPI validation workflows.
